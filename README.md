# Leadping Go SDK

Typed Go client source for the Leadping API, generated from the public Leadping OpenAPI document with Microsoft Kiota.

This SDK gives Go applications a fluent, strongly typed way to call Leadping API resources such as leads, conversations, SMS, phone numbers, automations, reports, billing, and users.

## At a glance

| Item             | Value                               |
| ---------------- | ----------------------------------- |
| Repository       | `leadpingai/leadping-go`            |
| Module path      | `github.com/leadpingai/leadping-go` |
| Package name     | `leadpinggo`                        |
| Main client      | `LeadpingOpenApiClient`             |
| Default API host | `https://api.leadping.ai`           |
| OpenAPI source   | `leadpingai/openapi`                |
| Generator        | Microsoft Kiota `1.32.2`            |

## Status

This repository contains generated SDK source plus Go module metadata. The `Release Go SDK` GitHub Actions workflow tests the module, tags the commit, and creates a GitHub release. That tag is the package publication mechanism for Go module consumers.

The generated source is refreshed from `leadpingai/openapi`. Do not hand-edit generated `.go` files; update the OpenAPI document and regenerate the SDK instead. This README is intended to be preserved across regenerations.

## Install

Install a released version with `go get`:

```bash
go get github.com/leadpingai/leadping-go@v0.1.0
```

Applications using the generated client also need a Kiota request adapter. The standard HTTP adapter can be added with:

```bash
go get github.com/microsoft/kiota-http-go
```

## Quick start

```go
package main

import (
    "context"
    "fmt"
    "os"

    leadping "github.com/leadpingai/leadping-go"
    abstractions "github.com/microsoft/kiota-abstractions-go"
    kiotahttp "github.com/microsoft/kiota-http-go"
)

type bearerTokenAuthenticationProvider struct {
    token string
}

func (p bearerTokenAuthenticationProvider) AuthenticateRequest(
    ctx context.Context,
    request *abstractions.RequestInformation,
    additionalAuthenticationContext map[string]interface{},
) error {
    request.Headers.Add("Authorization", "Bearer "+p.token)
    return nil
}

func main() {
    token := os.Getenv("LEADPING_API_TOKEN")
    if token == "" {
        panic("LEADPING_API_TOKEN is required")
    }

    adapter, err := kiotahttp.NewNetHttpRequestAdapter(
        bearerTokenAuthenticationProvider{token: token},
    )
    if err != nil {
        panic(err)
    }

    client := leadping.NewLeadpingOpenApiClient(adapter)

    me, err := client.Users().Me().Get(context.Background(), nil)
    if err != nil {
        panic(err)
    }

    fmt.Println(me.GetEmail())
}
```

For production applications, use an authentication provider that matches your Leadping credential flow and refreshes credentials as needed.

## Common entry points

The client exposes request builders for the main Leadping API areas:

| Area              | Builder                                                                                  |
| ----------------- | ---------------------------------------------------------------------------------------- |
| Leads             | `client.Leads()`                                                                         |
| Conversations     | `client.Conversations()`                                                                 |
| SMS               | `client.Sms()`                                                                           |
| Phone numbers     | `client.PhoneNumbers()`                                                                  |
| Automations       | `client.Automations()`                                                                   |
| Sources           | `client.Sources()`                                                                       |
| Reports           | `client.Reports()`                                                                       |
| Users             | `client.Users()`                                                                         |
| Businesses        | `client.Businesses()`                                                                    |
| Usage and billing | `client.Usage()`, `client.Wallets()`, `client.Transactions()`, `client.PaymentMethods()` |

Each builder mirrors the OpenAPI path structure and provides typed request and response models from the `models` package.

## Regenerating

This SDK is regenerated by the Leadping GitHub workflow from the canonical OpenAPI repository:

```bash
kiota generate \
  --openapi ../openapi/openapi.json \
  --language Go \
  --output . \
  --class-name LeadpingOpenApiClient \
  --namespace-name github.com/leadpingai/leadping-go \
  --exclude-backward-compatible
```
